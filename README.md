# adamduhachek.com

Personal academic website for **Adam Duhachek** — Bielinski Family Faculty Scholar & Professor of Marketing, University of Illinois Chicago.

A **Next.js 16** (App Router) app in **TypeScript**, deployed on **Vercel** at **adamduhachek.com**. The architecture mirrors the companion [well-being-atlas](https://github.com/adamduhachek-tech/well-being-atlas): a manifest-driven gallery, self-contained data essays served untouched inside an `<iframe>` reader, and generated Open Graph share cards — adapted here to read a committed local manifest instead of an object-storage bucket, since every essay ships with the repo.

## Stack

- **Next.js 16** App Router, **React 19**, **TypeScript**
- No CSS framework — a single hand-authored `globals.css` (the original academic palette, plus dark-mode support)
- Generated OG images via `next/og` (Satori)
- No runtime network or filesystem access: the gallery reads a committed JSON manifest, so everything is known at build time

## Structure

| Path | What it is |
|---|---|
| `src/app/(main)/page.tsx` | Home — the data-essay gallery (filter chips + search) |
| `src/app/(main)/about/page.tsx` | About — bio, title, contact, research interests |
| `src/app/(main)/research/page.tsx` | Research — overview + full curriculum vitae (51 publications) |
| `src/app/(main)/teaching/page.tsx` | Teaching — courses, international teaching, doctoral advising |
| `src/app/(main)/layout.tsx` | Shared chrome (nav + footer) for the four content pages |
| `src/app/essays/[slug]/page.tsx` | Essay reader — frames the essay with a back link + share bar |
| `src/app/opengraph-image.tsx`, `src/app/essays/[slug]/opengraph-image.tsx` | Generated share cards |
| `src/app/sitemap.ts`, `src/app/robots.ts` | SEO routes |
| `src/components/` | `SiteNav`, `SiteFooter`, `GalleryGrid`, `ShareBar` |
| `src/lib/` | `essays` (manifest loader), `gallery` (view model), `taxonomy` (tags), `sparkmark` (generated thumbnails), `site`, `og` |
| `src/data/essays.json` | **Curated source** manifest (title, dek, dataset, construct, primary viz per essay) — hand-editable |
| `src/data/manifest.json` | **Generated** manifest (adds reading time + inferred tags) — read by the app, regenerated by `npm run manifest` |
| `src/app/globals.css` | All styles |
| `public/essays/<slug>/index.html` | The 50 self-contained interactive data essays (hand-built D3, inlined data) |
| `public/Duhachek June 2026 CV.pdf` | CV (linked from Research) |
| `scripts/build-manifest.ts` | Regenerates `manifest.json` from `essays.json` + the essay HTML |

Each essay is an untouched, self-contained HTML document (it imports D3 from a CDN and inlines its own data). Essays live under `public/essays/<slug>/index.html` so their sibling cross-links (`../<other-slug>/index.html`) resolve, and are framed by the `/essays/[slug]` reader.

## Develop

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build
npm run lint
```

## Adding or editing an essay

1. Drop the self-contained HTML at `public/essays/<slug>/index.html`.
2. Add an entry to `src/data/essays.json` (`slug`, `title`, `dek`, `dataset`, `construct`, `primaryViz`).
3. Run `npm run manifest` to regenerate `src/data/manifest.json` (reading time + topic/scope/source tags).
4. Commit both files. The gallery, essay route, sitemap, and OG card pick it up automatically.

The gallery shows essays in the curated order of `essays.json`. Tags are inferred from the shared vocabulary in `src/lib/taxonomy.ts` and can be hand-corrected in `manifest.json`.

## Deploy on Vercel

1. Import this GitHub repo at [vercel.com/new](https://vercel.com/new). Vercel auto-detects Next.js — no build settings to change.
2. Deploy.

### Custom domain (adamduhachek.com)

In the Vercel project → **Settings → Domains**, add `adamduhachek.com` (and `www.adamduhachek.com`), then set the DNS records Vercel shows at your registrar:

- Apex `adamduhachek.com` → **A** record to the value Vercel displays (typically `76.76.21.21`), and/or
- `www` → **CNAME** to `cname.vercel-dns.com`.

Vercel issues the TLS certificate automatically once DNS resolves. On the production deployment the canonical URL and share cards use `https://adamduhachek.com` (set in `src/lib/site.ts`); preview deployments use their own Vercel URL.
